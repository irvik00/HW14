# Threat Analyzer

Автоматизированный мониторинг и реагирование на угрозы (учебный проект).

## Цель
Скрипт собирает данные из двух источников (API Vulners и логи Suricata), анализирует их на наличие критических уязвимостей и подозрительной активности, имитирует реагирование и формирует отчёт с визуализацией.

## Используемые источники
- **Vulners API** — получение информации об уязвимостях (CVE, CVSS).
- **Логи Suricata** (формат JSON lines, `eve.json`) — события сетевой безопасности.

## Установка и настройка

1. **Клонируйте репозиторий** (или создайте папку проекта вручную).

2. **Создайте и активируйте виртуальное окружение** (рекомендуется Python 3.8+):
   ```bash
   python -m venv venv
   source venv/bin/activate      # Linux/macOS
   venv\Scripts\activate         # Windows
   ```
   
3. **Установите зависимости:**

```bash
pip install -r requirements.txt
```

4. **Получите API ключ Vulners:**

- Зарегистрируйтесь на vulners.com.

- Перейдите в настройки профиля → «API Keys» → создайте новый ключ.

- Скопируйте его.


5. **Создайте файл ```.env``` в корне проекта и добавьте:**

```text
VULNERS_API_KEY=ваш_ключ_без_кавычек
```
6. **Подготовьте тестовые логи Suricata:**

- Поместите пример файла ```suricata_sample.json``` в папку ```logs/```.

- Формат: одна запись в строке (JSON lines). Пример содержимого можно найти в документации Suricata или сгенерировать самостоятельно (см. раздел «Тестовые данные»).

7. **Запустите скрипт:**

```bash
python main.py
```
## Результаты работы
После выполнения в папке ```reports/``` появятся:

- ```critical_cves.csv``` — список критических уязвимостей (CVSS >= 7.0).

- ```top_ips.csv``` — топ-5 IP-адресов по количеству срабатываний.

- ```summary.json``` — краткая статистика.

- ```threat_analysis.png``` — график (топ IP + распределение CVSS).

В консоль будет выведена информация о найденных угрозах и имитации блокировки.
